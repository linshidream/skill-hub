# 扩散收敛法 (diverge-converge)

把一个看不清全貌的不成熟想法,通过**先扩散、后收敛**,逐步逼近成一份可实施的手稿。

## 一句话

> 像和专家酣畅淋漓地聊一个想法——他帮你扩散出你不知道的点,你选方向往深里钻,每轮锁定一个决策,直到所有问题都解决完,MVP 方案浮现;最后一句"把以上聊天形成可实施文档",得到一份能交给其他智能体接手的手稿。

## 方法内核

```
理解想法 → [扩散 ↔ 选择深入 ↔ 收敛](循环,重心从扩散滑向收敛)→ 检测闭合 → 结晶成手稿
```

- **扩散**:铺开选项地图、暴露"未知的未知"、列权衡、标置信度、每轮只问一个最承重的问题。
- **收敛**:每轮锁定一个决策、落进"定版坐标"账本、量化阈值、记录被否决项、为未来留接口。
- **结晶**:把整段探讨忠实写成手稿——决策、依据、置信度、否决项、延后口子,一个不丢。

## 何时触发

- 用户说"先扩散后收敛"、"扩散收敛一下"、"diverge-converge"
- 用户带半成熟想法,想在动手前看清全貌
- 用户想把多轮探讨结晶成可实施文档 / 会议纪要 / 手稿

## 适用范围

**领域无关的通用思维方法,不是软件 skill。** 软件构建只是它最便于演示的场景之一。它同样适用于:研究选题、商业决策、产品设计、写作框架、政策方案、职业规划——任何**开放、复杂、没有现成答案、一开始看不清全貌**的课题。判断标准只有一条:你是否一开始看不清这件事的全貌?是,就用。

方法脱胎于成熟的思维传统——设计领域的**双钻模型(Double Diamond)**、心理学的**发散/收敛思维**(吉尔福特)——再配上 AI 协作:扩散借 AI 补认知盲区,收敛由人拍板定方向。

## 文件

- `SKILL.md` — 协议主体(四相循环、扩散/收敛的认知动作、纪律、反模式、跨领域适用)
- `templates/manuscript.md` — 结晶手稿骨架(领域中立,软件/研究/商业通用)
- `skill.json` — 元数据

## 定位:思维层,独立通用

本 skill 只负责**想清楚一件事**,产出一份手稿,完全独立、不依赖任何其他 skill。想清楚之后"怎么执行"因领域而异:软件可衔接 `dev-spec` / `dev-lifecycle` 等开发流程 skill,研究可衔接文献/统计工具,商业可衔接项目管理——这些都只是手稿的**可选下游**,不是必需依赖。**先想清(本 skill),再执行(任何领域的执行层)。**
