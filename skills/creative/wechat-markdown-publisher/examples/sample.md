# 一篇技术文章的发布样例

> 这是一段开头摘要，用来说明文章主题和阅读收益。公众号排版需要让它有信息密度，但不要显得沉重。

[TOC]

## 一、为什么需要固定排版

技术文章里常见的元素包括 `inline code`、列表、引用、表格、代码块和图片。如果每次都临时调整样式，发布成本会非常高。

- 保持 Typora 写作方式不变。
- 输出适合微信公众号粘贴的富文本。
- 让视觉风格稳定，不依赖每次手调。

## 二、流程示例

一个稳定的发布流程可以写成：

```text
Markdown
  -> styled HTML
  -> preview
  -> copy to WeChat editor
```

也可以包含配置：

```yaml
theme: media-flat
toc: true
imageMode: copied
```

## 三、表格和图片

| 元素 | 第一版策略 | 备注 |
| --- | --- | --- |
| 图片 | 复制本地图片，保留远程图片 | 粘贴后抽查 |
| 公式 | 检测并报告 | 后续增强 |
| 脚注 | 保留并样式化 | 依赖解析器 |
